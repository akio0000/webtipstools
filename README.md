このプロジェクトは、日々の記録、議事録、参考情報を管理し、ローカル AI（Llama-server）と連携して「自分だけのナレッジベース」を構築するための Streamlit ウェブアプリケーションです。

## 🌟 主な機能

### 1. 統合データベース管理 (SQLite)
- **複数人同時利用**: JSONからSQLiteデータベースへ移行したことで、複数ユーザーによる同時書き込みの安定性が大幅に向上しました。
- **自動バックアップ**: アプリ起動時にデータベースのバックアップを自動作成（最新10件を保持）。万が一の操作ミスでも `backups` フォルダから復旧可能です。
- **参考データの統合**: 既存のJSONファイル（参考jsonフォルダ内など）を起動時に自動インポートし、過去の資産をそのまま活用できます。

### 2. 記録の登録と動画最適化 (`register.py`)
- **マルチメディア対応**: 画像だけでなく、iPhone/Androidで撮影した動画（mp4, mov等）のアップロードに対応。
- **動画の自動最適化 (FFmpeg)**: 保存時に自動でリサイズ・圧縮（720p相当）を行い、ストレージ容量を節約しつつブラウザでの再生互換性を確保します。
- **AI 画像自動解析**: アップロードした画像を AI が自動解析し、内容をテキスト化。画像の中身も検索対象になります。

### 3. 高度な検索とプレビュー (`search.py`)
- **ソース別フィルタリング**: 「通常記録」「議事録」「参考データ」を切り替えて検索可能。
- **リッチプレビュー**: 画像、動画（プレイヤー再生）、PDF、Excel、CSVなどをその場で確認。
- **ドキュメント出力**: 検索結果や詳細内容を Word/Excel/PDF/ZIP 形式でエクスポート。

### 4. AI 相談ツール (RAG) (`ai_utils.py`)
- **ナレッジベース回答**: 登録された全データ（通常記録・議事録・参考資料）に基づき、ローカル AI が回答します。
- **キーワード自動抽出**: AI が質問文から重要な単語を抽出し、関連度の高い過去事例を背景知識として参照します。

## ⚙️ システム構成

- **Frontend**: Streamlit
- **Database**: SQLite 3 (ファイルベースDB)
- **AI Backend**: Llama-server (LM Studio / llama.cpp 等)
- **External Tools**: FFmpeg (動画の圧縮・変換に必須)

## 🚀 セットアップ

### 1. 接続設定
`config.py` を編集して、データ保存先と AI の接続先を指定します。

```python
# データ保存先（共有フォルダ等）
DEFAULT_DATA_DIR = r"\\network-server\save-data\records"

# AI接続先
AI_BASE_URL = "http://localhost:1234/v1"
```

### 2. 外部ツールの準備
- **FFmpeg**: 動画機能を利用する場合、インストールして環境変数 PATH を通してください。
- **Llama-server**: LM Studio 等を起動し、API サーバーを有効にしてください。

### 3. ライブラリのインストール
```bash
pip install streamlit pandas openai Pillow openpyxl python-docx fpdf2 qrcode
```

## ⚠️ 注意事項
- **安全な削除**: レコードを削除しても、サーバー上の物理ファイル（画像・動画）は削除されません。これにより誤操作によるデータ消失を防いでいます。
- **アップロード上限**: 1ファイルあたり最大 **1GB** まで対応しています（`.streamlit/config.toml` で変更可能）。

## 💡 使い方のヒント
- **Windows「送る」メニューへの登録**: 
  `shell:sendto` で開くフォルダに、起動用バッチファイル等のショートカットを作成して入れると、ファイル右クリックから素早く登録・検索ができます。
- **共有サーバーでの利用**: `config.py` で共有フォルダのパスを指定することで、チーム全体で一つのデータベースを共有・成長させることができます。

### 🔄 他アプリからのデータ移行（吸収機能）
過去に別のアプリ等で作成したJSON形式のデータを、当システムのデータベースに一括で取り込むことができます。

1. `config.py` 内の `REFERENCE_JSON_DIR` に、取り込みたいJSONファイルが入っているフォルダのパスを指定します。
   ```python
   # ネットワークサーバーを指定する場合の例
   REFERENCE_JSON_DIR = Path(r"\\SERVER_NAME\SharedFolder\ReferenceData")
   ```
2. アプリを起動（またはブラウザをリロード）すると、指定したフォルダ内のJSONファイルが自動的にデータベースへ取り込まれます。
3. **重複防止**: すでに取り込み済みのデータは自動でスキップされるため、何度起動しても二重登録される心配はありません。
4. **段階的な取り込み**: パスを変更してアプリを起動する手順を繰り返せば、複数のフォルダから次々とデータを「吸収」していくことができます。

> [!WARNING]
> **画像などの添付ファイルに関する注意**
> 取り込むJSONデータに画像などの添付ファイルパスが含まれている場合、データベースにはその「ファイルの場所（パス）」が記録されます。そのため、**移行後に元の画像ファイル自体を移動・削除してしまうと、アプリ上からプレビューできなくなります**。テキストデータのみの場合は影響ありません。
